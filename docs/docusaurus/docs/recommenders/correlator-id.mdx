# Tracking recommendations outcomes using Correlator Id

Explicitly linking events to a recommendation will improve the accuracy of the recommender, and the speed at which it learns.

Every recommendation comes with a `correlatorId`

```json
{
  ... // recommendation
  "correlatorId": 104346873
}
```

signalbox.events.logUserEvents({

      recommendationCorrelatorId: 104346873, //
      commonUserId: "test-common-user-id",
      eventId: "unique event Id", // required
      timestamp: new Date(), // optional - defaults to current time.
      kind: "Purchases", // required - high level categorisation of event
      eventType: "ItemPurchased", //required - specific categorisation of event
      properties: {
        value: 33.6,
      },
    },
  ],
});
